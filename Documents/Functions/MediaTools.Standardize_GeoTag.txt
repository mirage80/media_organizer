📍 MediaTools.Standardize_GeoTag Function Documentation

Function:
Standardize_GeoTag

Description:
Converts various GPS coordinate formats into a consistent decimal degrees representation with proper sign based on hemisphere.

Input Type:
1. $latitude  [string] — Raw latitude string (e.g., "37.7749 N", "37°46'29.64\"N")
2. $longitude [string] — Raw longitude string (e.g., "122.4194 W", "122°25'9.84\"W")

Output Type:
✅ [hashtable] — Contains keys "Latitude" and "Longitude" with standardized decimal values
❌ [null] — If input is missing or cannot be parsed

Algorithm (Pseudo-code):
1. Validate inputs
   → If either $latitude or $longitude is null or empty → return null

2. Normalize strings
   a. Remove spaces, quotes, and degree symbols
   b. Detect hemisphere suffix (N/S/E/W)

3. Split coordinates into components
   a. If DMS format (degrees, minutes, seconds):
      - Convert to decimal using:
        decimal = degrees + (minutes / 60) + (seconds / 3600)
   b. If already decimal:
      - Parse directly

4. Apply sign:
   → S/W → negative
   → N/E → positive

5. Return hashtable:
   @{
     Latitude = <decimal_lat>
     Longitude = <decimal_lng>
   }

Function Calls:
🔹 MediaTools.ParseToDecimal — Used for DMS to decimal conversion
🔹 MediaTools.Normalize_Directions_In_String — Normalizes directional suffixes

Possible Issues:
✅ Covered: N/S/E/W suffix handling
✅ Covered: DMS and decimal format parsing
❌ Not Covered: Invalid mixed-format input (e.g., "37.77, -122W")
❌ Not Covered: Latitude/longitude out of bounds
❌ Not Covered: Empty direction indicators
