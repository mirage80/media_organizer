🕒 MediaTools.Standardize_TimeStamp Function Documentation

Function:
Standardize_TimeStamp

Description:
Normalizes various timestamp formats into a consistent pipeline format: "yyyy:MM:dd HH:mm:ss+00:00"

Input Type:
1. $timestamp [string] — Raw timestamp string from metadata

Output Type:
✅ [string] — Standardized timestamp in UTC ISO format
❌ [null] — If the timestamp is missing or invalid

Algorithm (Pseudo-code):
1. If $timestamp is null or empty
   → Return null

2. Remove unwanted characters (e.g., enclosing quotes, trailing whitespace)

3. Detect common formats:
   a. ISO8601: 2023-12-01T15:30:00Z
   b. EXIF:   2023:12:01 15:30:00
   c. Custom: 2023/12/01 15:30, etc.

4. Parse using Get-Date or custom regex
   → If parse succeeds:
       a. Convert to UTC
       b. Format as "yyyy:MM:dd HH:mm:ss+00:00"
       c. Return result

5. If parse fails:
   → Return null

Function Calls:
🔹 MediaTools.Parse_DateTimeFromFilename — Attempts parsing timestamps from filenames
🔹 Get-Date — Used to parse and format standard timestamps

Possible Issues:
✅ Covered: EXIF and ISO timestamp patterns
✅ Covered: Extra characters and whitespace
❌ Not Covered: Locale-dependent formats (e.g., MM/DD/YYYY)
❌ Not Covered: Inconsistent or partial timestamps (missing seconds or year)
❌ Not Covered: Conflicting timezones without standard suffix
