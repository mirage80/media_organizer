📁 MediaTools.New-GeoTagFromParts Function Documentation

**Description:**  
Constructs a new geotag string by modifying selected components (latitude or longitude) of an existing geotag string. This function allows fine-grained updates to individual geotag fields, returning a valid ISO6709-style GPS coordinate string.

---

**Input Type:**  
1. `current_geotag [string]` — The current geotag string to be modified (e.g., "37.7749,N,122.4194,W").  
2. `fields [string[]]` — The field names to update (e.g., ["latitude", "longitude"]).  
3. `values [string[]]` — The new values to assign to the corresponding fields. Must match the length of `fields`.

---

**Output Type:**  
✅ `[string]` — A valid geotag string with updated values (e.g., "48.8566,N,2.3522,E").  
❌ Returns `null` or throws on failure if invalid input is supplied.

---

**Algorithm (Pseudo-code):**  
1. Parse `current_geotag` into its four components: latitude, latitudeRef, longitude, longitudeRef.  
2. Iterate through `fields`, replacing matching parts with new values from `values`.  
3. Reconstruct the final geotag string in the format: "<lat>,<latRef>,<lon>,<lonRef>".  
4. Return the updated geotag string.

---

**Function Calls:**  
🔹 Get-LatitudeReference  
🔹 Get-LongitudeReference

---

**Possible Issues:**  
✅ Covered: Handles direction (N/S, E/W) by calling helper reference functions.  
✅ Covered: Returns null on malformed input or invalid keys.  
❌ Not Covered: No strict validation of numeric bounds for coordinates.  
❌ Not Covered: Does not auto-normalize directional conflicts (e.g., 90.0,S).
