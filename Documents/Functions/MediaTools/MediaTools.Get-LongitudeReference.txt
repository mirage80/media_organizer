🧭 MediaTools.Get-LongitudeReference Function Documentation

Description:
Determines the correct longitude reference ("E" for East, "W" for West) from a signed decimal longitude value.

Input Type:
1. longitude [double] (Mandatory) — The signed decimal longitude value (e.g., `-74.0060`, `151.2093`).


Output Type:
✅ [string] — Returns "E" if the longitude is positive or zero. Returns "W" if the longitude is negative.

Algorithm (Pseudo-code):
1. Accept a mandatory `[double]` parameter named `$longitude`.
2. If `$longitude` is greater than or equal to 0, return "E".
3. Otherwise, return "W".

Function Calls:
None.

Possible Issues:
* ✅ Covered: Correctly handles positive, negative, and zero values.
* ✅ Covered: The parameter is mandatory, which prevents `$null` input from being silently converted to `0` and returning an incorrect "E" reference.
* ❌ Not Covered: This function does not validate if the longitude value is within the standard -180 to +180 range. It only checks the sign.
