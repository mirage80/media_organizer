📂 MediaTools.Move-FileToDestination Function Documentation

Function:
Move-FileToDestination

Description:
Moves a media file to a destination path, creating directories if necessary and optionally overwriting existing files.

Input Type:
1. $source [string] — Full path to the file to be moved
2. $destination [string] — Full destination file path
3. $logFunc [scriptblock] — Optional logger callback to track operations
4. $overwrite [bool] — Whether to overwrite existing file (default: $false)

Output Type:
✅ [void] — File moved to destination
❌ Logs an error if file move fails

Algorithm (Pseudo-code):
1. Validate Paths
   a. If $source does not exist → log error, return
   b. If $destination's directory does not exist → create it

2. Handle Overwrite
   a. If $overwrite is $true and destination exists → delete it

3. Attempt Move
   a. Use Move-Item to move $source to $destination
   b. Catch exceptions → log warning

4. Logging
   a. If move successful → log info
   b. If move fails → log error

Function Calls:
🔹 Move-Item — Performs the file move
🔹 Test-Path — Validates file and directory existence
🔹 New-Item -ItemType Directory — Creates target folder if missing
🔹 Remove-Item — Deletes target file if overwriting

Possible Issues:
✅ Covered: Creates directory if missing
✅ Covered: Optional overwrite logic
✅ Covered: Logs both success and failure
❌ Not Covered: Race conditions in concurrent moves
❌ Not Covered: Permissions issues (read-only or locked files)
❌ Not Covered: Invalid characters in path names
