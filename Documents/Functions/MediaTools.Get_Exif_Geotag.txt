Function:
MediaTools.Get_Exif_Geotag

Description:
Extracts and returns a standardized GPS geotag from Exif metadata, using latitude and longitude fields retrieved from a file's Exif data.

Input Type:
1. ExifData [PSCustomObject] — Parsed JSON metadata object from ExifTool (e.g., from Get-ExifDataAsJson)

Output Type:
✅ [string] — GPS coordinate in ISO 6709 format (e.g., +37.7749-122.4194+00:00)
❌ Returns $null if required fields are missing or invalid

Algorithm (Pseudo-code):
1. Validate input ExifData
    1.1 If ExifData is null or missing required GPS fields → return $null
2. Extract GPSLatitude and GPSLongitude
    2.1 Also extract GPSLatitudeRef and GPSLongitudeRef if available
3. Normalize directions using MediaTools.Normalize_Directions_In_String
    — Purpose: Ensure directions are in standard N/S/E/W format
4. Convert coordinates to decimal using MediaTools.ParseToDecimal
    — Purpose: Convert degree-minute-second to decimal notation
5. Apply negative sign if needed based on direction refs (S, W)
6. Format decimal values into ISO 6709 string
    6.1 Append +00:00 as timezone offset
7. Return the formatted geotag string

Possible Issues:
✅ Covered: Missing GPSLatitude or GPSLongitude in ExifData
✅ Covered: Null input
✅ Covered: Non-decimal format handled via ParseToDecimal
✅ Covered: Directional inconsistency handled via Normalize_Directions_In_String
❌ Not Covered: Invalid coordinate range (e.g., lat > 90)
❌ Not Covered: Incorrect GPS timezone offset if not UTC
