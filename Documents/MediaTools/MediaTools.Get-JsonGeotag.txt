📍 MediaTools.Get-JsonGeotag Function Documentation

Description:
Extracts the geotag from a Google Photos `.json` sidecar file and returns it as a standardized 4-part geotag string.

Input Type:
1. JsonFile [System.IO.FileInfo] (Mandatory) — The `.json` sidecar file to process.

Output Type:
✅ [string] — The standardized 4-part geotag string.
❌ Returns `$null` if the `geoData` field is missing or incomplete.
❌ Throws an exception if the found geotag is invalid and cannot be standardized.

Algorithm:
1. Read and parse the input `JsonFile`.
2. Extract the `latitude` and `longitude` from the `geoData` property.
3. If both values exist, combine them into a 2-part string (e.g., `"40.7128, -74.0060"`) and pass it to `Format-GeoTag`.
4. Return the result from `Format-GeoTag`.
5. If the `geoData` is missing or incomplete, return `$null`.

Function Calls:
🔹 Format-GeoTag

Possible Issues:
* ✅ Covered: Gracefully handles missing or incomplete `geoData` fields.
* ✅ Covered: Ensures all output is in the module's canonical format by delegating parsing to `Format-GeoTag`.
* ❌ Not Covered: This function specifically targets the `geoData.latitude` and `geoData.longitude` fields and will not find geotags stored elsewhere in the JSON file.
