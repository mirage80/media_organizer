📁 MediaTools.Get-FilenameTimestamp Function Documentation

Description:
Extracts a timestamp from a file's name by matching against common embedded date/time patterns. If a valid pattern is found, it converts the substring to a [DateTime] object. Returns $null if no match or parsing fails.

Input Type:
1. FilePath [System.IO.FileInfo] — The media file whose filename will be parsed for a timestamp.

Output Type:
✅ [nullable DateTime] — A valid timestamp if one is found in the filename.
❌ Returns $null if no match or parsing fails.

Algorithm (Pseudo-code):
1. Use regex to match timestamp patterns embedded in the filename.
2. If a pattern matches, parse it into a DateTime object using exact format.
3. Return the DateTime object or $null if parsing fails.

Possible Formats:
a. yyyy-MM-dd_HH-mm-ss  
b. yyyyMMdd_HHmmss  
c. yyyyMMdd

Function Calls:
✅ None (pure logic)

Possible Issues:
✅ Regex mismatch may result in false negatives  
✅ Returns $null silently on all errors  
❌ Not Covered: No fallback for ambiguous or overlapping date formats
